# AI Race Engineer — Intelligent Telemetry Coaching for iRacing GT3

An end-to-end AI coaching system for iRacing sim-racing. It downloads real lap telemetry from [Garage61.net](https://garage61.net), trains an LSTM Autoencoder on expert laps, and compares any uploaded amateur lap against the learned expert pattern — identifying where time is lost and why, with natural-language coaching generated by an LLM.

---

## Overview

```
Garage61 CSVs  →  Data Pipeline  →  LSTM Autoencoder  →  Statistical Analysis  →  LLM Feedback  →  Streamlit App
```

The system has three layers of feedback:

| Layer | Method | Output |
|---|---|---|
| 1 | Statistical (deterministic) | Zones, severity scores, causal chains |
| 2 | Template rendering (deterministic fallback) | Structured coaching text |
| 3 | LLM via Groq | Polished natural-language coaching |

---

## Features

- **Automated data collection**: downloads laps from Garage61 using Playwright (browser automation + HTTP interception)
- **LSTM Autoencoder**: trained on expert telemetry; reconstruction error = deviation from expert driving style
- **Zone detection**: segments the lap into problem zones ranked by severity (0–1 score)
- **Causal chain analysis**: detects cause → effect relationships between channels (e.g. late brake → low speed → understeer)
- **Progressive Streamlit UI**: 3-phase pipeline renders results as they arrive without blocking:
  - Phase 1 (instant): metadata, raw telemetry
  - Phase 2 (~2–5 s): anomaly heatmap, telemetry overlays, track map
  - Phase 3 (background): LLM coaching cards
- **Track map**: circuit geometry from GPS lat/lon, with coloured zone markers
- **Multi-page app**: Session Overview, Telemetry Analysis, Session Report, Advanced

## Setup

### 1. Prerequisites

- Python 3.10+
- A [Garage61.net](https://garage61.net) account (for downloading laps)
- A [Groq](https://console.groq.com) API key

### 2. Install dependencies

```bash
python -m venv venv
venv\Scripts\activate          # Windows

pip install -r requirements.txt
playwright install chromium
```

### 3. Configure API keys

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
# GEMINI_API_KEY=your_google_api_key_here   # optional, for Gemini
```

## Usage

### Run the Streamlit app

```bash
streamlit run app/main.py
```

Upload a Garage61 CSV from the sidebar. The app will automatically detect the driver name and lap time from the filename.

---

### Download laps from Garage61

```bash
python garage61_downloader.py
```

This opens a Chromium browser for manual login, then automatically:
1. Navigates to the configured track/car filter
2. Scrolls to load all laps
3. Downloads CSV for each public lap

Configure `TRACK_ID` and `CAR_ID` at the top of `garage61_downloader.py`.

---

### Train the autoencoder

```bash
python train_autoencoder.py
```

Trains a sequence-to-sequence LSTM Autoencoder on all processed expert laps. Saves `autoencoder_best.pt` and `scaler_params.npz` to `models/`.

Key hyperparameters (in `config.py`):

| Parameter | Value |
|---|---|
| Architecture | LSTM Autoencoder |
| Hidden size | 256 |
| Latent dim | 32 |
| LSTM layers | 2 |
| Sequence length | 1000 points/lap |
| Features | 9 telemetry channels |
| Noise augmentation | 8× per lap |

---

### Coach a specific pilot (CLI)

```bash
python coach_pilot.py
```

Point `PILOT_DIR` inside the script to the folder containing the pilot's Garage61 CSVs. Generates telemetry overlay plots and a JSON coaching report in `models/eval_*/`.

---

### Hyperparameter optimisation

```bash
python optimize_hyperparams.py
```

Runs an [Optuna](https://optuna.org) search over dropout, noise std, learning rate, batch size, and weight decay. Results saved to `models/best_hyperparameters.json`.

## Model Architecture

```
Input: (batch, 1000, 9)   — 1000 lap points × 9 telemetry channels
         ↓
Encoder LSTM (hidden=256, layers=2)
         ↓
Latent vector (32,)
         ↓
Decoder LSTM (hidden=256, layers=2)
         ↓
Output: (batch, 1000, 9)  — reconstructed expert-style telemetry
```

Reconstruction error (MSE per point) reveals where the uploaded lap deviates from the expert pattern. High-error segments become **coaching zones**.

---

## Coaching Zone Pipeline

1. **Compute signed error**: `recon - original` in physical units for each channel
2. **Detect zones**: contiguous segments above the MSE threshold
3. **Analyse channels**: rank channels by deviation within each zone
4. **Detect causal chains**: identify temporal cause → effect relationships (e.g. early throttle → understeer → wide exit)
5. **Score severity**: weighted 0–1 score per zone
6. **Estimate time loss**: approximated from speed deviation integral
7. **Generate feedback**: template text first, then enriched by LLM if available

## Data Flow

```

    │
    ▼
interpolate_lap()          # Resample to 1000 points on LapDistPct axis
    │
    ▼
apply_normalization()      # Z-score per channel using scaler_params.npz
    │
    ▼
LSTMAutoencoder.forward()  # Reconstruct expert-style telemetry
    │
    ▼
reconstruct_errors()       # Per-point MSE
    │
    ▼
detect_zones()             # Problem segment detection
    │
    ▼
generate_feedback()        # Statistical → Template → LLM
    │
    ▼
Streamlit pages            # Progressive rendering
```
